# 城市機車友善地圖系統 - 產品需求文件 (PRD)

## 1. 專案概述
### 背景與動機
在台灣，機車是極度普遍的代步工具，然而許多城市的道路設計或路況往往對機車騎士充滿潛在危險。許多新手騎士或外地通勤者因為不熟悉路況而發生意外。為了提升騎士的行車安全，我們計畫開發「最強最安全的城市機車友善地圖系統」，提供針對機車族群量身打造的交通地圖與安全警示。

### 目標用戶
- 大學生機車族（常在學區與宿舍間移動，可能對路況較陌生）
- 剛考到駕照的新手騎士（缺乏實際道路經驗，需要更多安全指引）
- 日常通勤的機車使用者（需要掌握每日通勤路段的最新路況與潛在危險）

### 核心價值主張
透過群眾力量與地理資訊，建立針對機車族群的危險路段評價系統，提供即時、精準的安全路況預警，降低事故發生率，打造最友善的城市騎乘環境。

## 2. 功能需求

以下列出本系統的主要功能，其中重點聚焦於你所負責的「危險路段評價系統」。

### 2.1 危險路段評價系統 (核心功能)
- **使用者故事**：作為一名日常通勤的機車使用者，我希望能在地圖上標記曾經發生打滑或視線死角的危險路段，以便提醒其他騎士注意。
- **使用者故事**：作為一名新手騎士，我希望在查詢路線時能看到即將經過的危險路段評價與星級，以便我能提前減速或避開該路段。
- **使用者故事**：作為一名大學生機車族，我希望能對其他使用者標記的危險路段進行「贊同」或「評論」，以便提升該警示的準確度與參考價值。

### 2.2 機車友善地圖瀏覽
- **使用者故事**：作為一名機車使用者，我希望能在地圖上清楚看到機車專用道、機車待轉區的位置，以便我能正確且合法地行駛。

### 2.3 路線導航與避雷建議
- **使用者故事**：作為一名新手騎士，我希望系統能提供「最安全」的路線導航（避開高危險評價路段），以便我能安心到達目的地。

### 2.4 即時路況回報
- **使用者故事**：作為一名通勤者，我希望可以即時回報路面坑洞、施工或臨停佔道，以便其他騎士能即時掌握突發狀況。

### 2.5 個人化騎乘紀錄與成就
- **使用者故事**：作為一名大學生，我希望系統能記錄我的安全騎乘里程並給予成就徽章，以便鼓勵我持續安全駕駛並積極參與回報。

### 2.6 附近友善與安全推薦功能
- **使用者故事**：作為一名機車使用者，我希望能在地圖或列表上看到當前位置或目的地附近的機車行、加油站、換電站，以便在車子故障或油量/電量不足時能快速獲取幫助。
- **使用者故事**：作為一名新手騎士，我希望在搜尋附近推薦時，能看到附近高風險危險路段的警示，以便我在行經時提前減速或選擇避開。
- **使用者故事**：作為一名通勤者，我希望可以查詢附近的機車停車場，以便我能安全合法地停放車輛。

## 3. 非功能需求
### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite

### 效能與安全考量
- 系統需在合理的延遲時間內載入地圖及危險標記。
- 使用者回報的危險路段需經過基礎的防呆機制（如避免短時間內重複回報同一地點），防止惡意洗版。
- 保護使用者隱私，匿名化回報者的個資，確保資料安全。

## 4. MVP 範圍 (最小可行性產品)

### Must Have (必須具備)
- 危險路段標記與評價功能（使用者可以在地圖特定點位新增危險路段，並輸入描述與危險星級）。
- 危險路段資訊顯示（地圖上能顯示各危險點，點擊可查看詳細資訊及評價）。
- 使用者註冊與登入（基本帳號管理以追蹤回報來源）。

### Should Have (應該具備)
- 危險點的留言與「贊同/反對」投票機制。
- 地圖清單檢視模式（列表顯示附近的危險路段）。
- 簡單的熱點圖顯示（用顏色區分危險程度高的區域）。
- 附近的機車友善點位（機車行、加油站、換電站、停車場）清單與地圖標記功能。
- 結合 HTML5 定位與距離排序推薦（預設方圓 2 公里內，依距離由近到遠排序，且可篩選類別）。

### Nice to Have (最好具備)
- 避開危險路段的路線導航功能。
- 結合政府開放資料（如即時交通事故資料）。
- 騎乘安全成就系統與排行榜。
